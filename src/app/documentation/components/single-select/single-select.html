How DomSingleSelectComponent handles searching, selection, and clear.

Selection flow

1

Panel opens

User clicks field or chevron. filteredOptions shows all options.

→
2

Search filters

searchQuery signal drives filteredOptions computed on title_property.

→
3

User selects

onSelectionChange emits the full option object T via on_change.

→
4

Panel closes

searchQuery cleared. Clear ✕ button appears. Control value = id_property value.

on_change payload note

on_change emits the full T object — not the stored ID.

The form control value is option[id_property] (e.g. "us"). But on_change emits the matching option object (e.g. { id: "us", title: "United States" }) — useful for displaying the selected item label without another options lookup.
