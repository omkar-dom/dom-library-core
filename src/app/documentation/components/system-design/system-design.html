1. Core Architecture & Technologies

Framework layers and dependency stack configurations

DOM Library Core is designed inside a monorepo workspace containing an application layer (the documentation website showcase) and a compiled, distribution-ready control library. The pipeline utilizes modern web features to keep assets lightweight and lightning-fast.

Angular 21 (Zoneless Signals)

Change detection runs completely without Zone.js overhead. State changes are managed using high-precision Angular Signals, giving 0ms detection delay.

Tailwind CSS v4 Integration

Tailwind v4 is configured with PostCSS CSS-first parsing. Built-in library utilities scan components using @source path directives to minimize the production footprint.

Standalone Design Pattern

Every control component is self-contained. Dependencies are imported locally in the component decorator, completely tree-shaking unused elements.

ESBuild & Vite Runtime

The local development server is powered by Vite. Distribution packaging is compiled cleanly via @angular/build:application and ng-packagr builders.

4. GitHub Production Hosting & CI/CD

Automated pipeline workflows and hosting config workarounds

The production website is hosted using **GitHub Pages** with fully automated CI/CD pipelines configured via GitHub Actions.

Workflow Pipeline Configuration

Managed inside .github/workflows/deploy.yml. Whenever changes are pushed to the main branch, a GitHub Actions runner installs Node.js, compiles the Angular codebase, and publishes built files directly to the gh-pages branch.

Deep Routing Workaround (404.html redirection)

Because GitHub Pages hosts static files, refreshing deep routes (like /documentation/install) results in a native 404 response. To solve this, during compilation our script copies the built index.html as 404.html inside the dist folder: 

cp dist/dom-library/browser/index.html dist/dom-library/browser/404.html

When GitHub Pages encounters an unrecognized path, it automatically falls back to 404.html. Since this file contains our complete Angular app, the runtime loads successfully and Angular Router routes the visitor correctly.

Workflow Permissions

Ensure write permissions are active in your GitHub Repository settings: Settings > Actions > General > Workflow permissions > Select 'Read and write permissions'.
