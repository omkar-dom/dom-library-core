File processing pipeline and control binding options for DomFilesComponent.

File processing pipeline

1

Files received

Via file picker click or drag-and-drop (onFilesSelected / onDrop)

→
2

Validated

Each file checked against max_size_mb and accept. First failure sets has_error and stops.

→
3

Stored

selected_files signal updated. FileReader generates preview_urls for images.

→
4

Synced

control().setValue(updated) + on_change.emit(updated) to notify the parent.

Control binding modes

Mode A — FormGroup + control_name

this.form = fb.group({ file: [null] });

<dom-files
  [form_group]="form"
  control_name="file"
/>

Best for forms with multiple controls — integrates with formGroup submit and reset.

Mode B — Direct FormControl

readonly file_ctrl = new FormControl(null);

<dom-files
  [form_control]="file_ctrl"
/>

Best for standalone uploads outside a FormGroup, or dynamically created controls.
