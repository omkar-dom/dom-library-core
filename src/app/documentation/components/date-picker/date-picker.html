Min/max validator architecture

How domDateMin / domDateMax connect to the Material calendar

  1. domDateMin("2024-01-01") creates a ValidatorFn with a domDateMin property attached.
  2. getDateBoundsFromControl() reads all control validators, finds validator.domDateMin / validator.domDateMax strings, and converts them to Date objects.
  3. The dateBounds computed signal feeds minDate() and maxDate() which are bound to the Material datepicker's [min] and [max] — greying out invalid calendar dates.
  4. The validator also runs synchronously and adds a domDateMin / domDateMax error key to the control, which <dom-error> displays when touched.
